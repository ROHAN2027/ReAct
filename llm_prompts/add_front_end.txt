# Premium Web Frontend for ReAct Agent

Build a stunning, dark-mode chat interface that connects to the existing ReAct agent via a FastAPI backend with Server-Sent Events (SSE) for real-time streaming.

## Architecture

```mermaid
flowchart LR
    A["Browser UI"] -->|POST /api/chat| B["FastAPI Server"]
    A -->|GET /api/chat/stream| B
    B --> C["LangGraph Agent"]
    C -->|tool calls| D["Tools + Memory"]
    D -->|observations| C
    C -->|streamed events| B
    B -->|SSE events| A
```

**Backend**: FastAPI wrapping the existing `create_agent()` + `save_interaction_to_memory()`. A single `/api/chat` endpoint accepts user messages and streams back events (thinking, tool calls, observations, final answer) as SSE.

**Frontend**: A single-page app with `index.html`, `style.css`, and `app.js` served from a `static/` directory. No framework — vanilla JS with modern CSS.

## Design Aesthetic

- **Dark mode** with deep navy/charcoal base (`#0a0a1a`, `#12122a`)
- **Glassmorphism** panels with `backdrop-filter: blur()` and subtle borders
- **Accent gradient**: Electric blue → purple (`#6366f1` → `#8b5cf6`)
- **Typography**: Inter from Google Fonts
- **Micro-animations**: Messages slide in, tool calls pulse, typing indicator with bouncing dots
- **Tool call visualizer**: Collapsible cards showing tool name, input, and output with syntax highlighting
- **Memory indicator**: A subtle glow/badge when the agent accesses long-term memory
- **Session panel**: Sidebar showing session ID and memory stats

## Proposed Changes

### Backend API

#### [NEW] [server.py](file:///home/divyansh/projects/dlProject/ReAct/server.py)

FastAPI application with:

- **`POST /api/chat`** — Accepts `{"message": "..."}`, runs the agent, streams back SSE events:
  - `event: thinking` — agent is reasoning
  - `event: tool_call` — `{name, args}` when a tool is invoked
  - `event: tool_result` — the observation from the tool
  - `event: answer` — the final response text
  - `event: memory_saved` — confirmation that the interaction was indexed
  - `event: error` — if something goes wrong
- **`GET /api/memory/stats`** — Returns memory stats (doc count, session count)
- **`GET /`** — Serves the static frontend
- Agent + memory_manager initialized once at startup
