# Long-Term Memory System for ReAct Agent

Add a persistent vector-store-backed memory system so the agent can store every conversation turn and retrieve relevant past context in future sessions.

## Architecture Overview

```mermaid
graph TD
    A[User Input] --> B[Agent Node]
    B -->|"References past sessions?"| C[SearchLongTermMemory Tool]
    C --> D[(ChromaDB<br>./agent_memory)]
    D --> C
    C --> B
    B --> E[Tool Execution<br>SafeToolNode]
    E --> B
    B --> F[Final Response]
    F --> G[Auto-Index to ChromaDB]
    G --> D
```

## Key Design Decisions

> [!IMPORTANT]
> **Embedding Model**: Using `sentence-transformers/all-MiniLM-L6-v2` instead of `google/embeddinggemma-300m`. Rationale: MiniLM-L6-v2 is a well-supported, lightweight (80MB) model that runs efficiently on CPU, has first-class integration with LangChain's `HuggingFaceEmbeddings`, and produces high-quality 384-dim embeddings perfect for conversational retrieval. EmbeddingGemma-300M is a much heavier download (~600MB+), has limited LangChain integration, and is overkill for storing/retrieving conversation snippets.

> [!IMPORTANT]
> **Vector DB**: Using **ChromaDB** with local persistence (`./agent_memory`). Chroma is simpler to set up than FAISS for persistent workloads because it natively supports `persist_directory` — no manual save/load calls needed. Documents are auto-persisted on every `add_documents()` call.

> [!NOTE]
> **Memory Module Pattern**: Instead of scattering vector store initialization across files, I'll create a dedicated `react_agent/memory.py` module that owns the ChromaDB client + embedding model lifecycle. Both `tools.py` and `agent_logic.py` will import from this module — single source of truth, no duplicate initialization.

### New Module: Memory Singleton

#### [NEW] [memory.py](file:///home/pranav/projects/ReAct/react_agent/memory.py)

Central module that owns the vector store lifecycle:

- **`get_vector_store()`** — Returns a singleton `Chroma` instance backed by `./agent_memory`. If the directory exists, it loads the existing collection; otherwise, it creates a new one.
- **`get_embeddings()`** — Returns a singleton `HuggingFaceEmbeddings` instance using `all-MiniLM-L6-v2`.
- **`save_interaction(user_input, agent_response, session_id)`** — Embeds and upserts a conversation turn as a document with metadata (`timestamp`, `session_id`, `turn_type`).
- **`search_memory(query, top_k=5)`** — Performs similarity search and returns the top-K most relevant past conversation snippets with metadata.

The collection name will be `"react_agent_memory"`.

### Memory Search Tool

#### [MODIFY] [tools.py](file:///home/pranav/projects/ReAct/react_agent/tools.py)

1. **New Pydantic schema** `LongTermMemoryInput` with a `query` field.
2. **New factory function** `create_long_term_memory_tool()` — Creates a `@tool("search_long_term_memory")` that calls `memory.search_memory(query)` and formats the results as a readable string with timestamps and session IDs.
3. **Register** the new tool in `get_all_tools()` alongside the existing tools.

---

### Auto-Indexing Logic

#### [MODIFY] [agent_logic.py](file:///home/pranav/projects/ReAct/react_agent/agent_logic.py)

1. **Import** `save_interaction` and `uuid4` at the top.
2. **Generate a `session_id`** in `create_agent()` (one per agent instantiation) using `uuid4().hex[:12]`.
3. **New `save_to_memory` node** added to the LangGraph after the agent produces a final answer (no more tool calls). This node:
   - Extracts the original user message and the final AI response from the state.
   - Calls `save_interaction(user_input, agent_response, session_id)`.
   - Returns the state unchanged (passthrough).
4. **Updated graph edges**: The conditional edge from `agent` now routes to either `tools` (if tool calls present) or `save_to_memory` → `END` (if final answer).
5. **Update `main.py`** to pass `session_id` into the REPL loop context (or let it be internally generated per `create_agent()` call).

Graph flow becomes:
```
START → agent ←→ tools
              → save_to_memory → END
```

### Updated Prompt

#### [MODIFY] [prompts.py](file:///home/pranav/projects/ReAct/prompts.py)

Add two new sections to `REACT_SYSTEM_PROMPT_TEMPLATE`:

1. **Long-Term Memory Awareness** — Instructs the agent to call `search_long_term_memory` when the user mentions "previously", "last time", "remember", "my preferences", past sessions, or any reference to prior context.

2. **Retrieved Context Section** — A `{retrieved_context}` placeholder in the template where pre-fetched memory can be injected. For the initial implementation, this will be empty (the agent uses the tool on-demand), but the placeholder enables future auto-retrieval.

3. **Tool Selection Logic Update** — Add `search_long_term_memory` to the tool list under a new "Memory" category.
